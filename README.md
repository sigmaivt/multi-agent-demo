# Multi-Agent Demo — конвейер «Исследователь → Писатель → Критик»

Учебный проект для изучения паттерна **оркестрации нескольких AI-агентов**.

Пользователь вводит тему → три агента последовательно создают и оценивают статью → оркестратор принимает решение о доработке.

## Архитектура

```
Пользователь → Оркестратор → [Исследователь] → [Писатель] → [Критик]
                     ↑                                           |
                     └──── если оценка < 7: доработка ──────────┘
```

**Ключевая идея:** агент — это не особый объект. Это обычный вызов LLM с уникальным `system` промптом, который задаёт роль. Оркестратор — это Python-код, который управляет потоком.

### Роли агентов

| Агент | Роль | Вход | Выход |
|-------|------|------|-------|
| Исследователь | Собирает ключевые тезисы по теме | Тема | Список тезисов |
| Писатель | Пишет статью на основе тезисов | Тема + тезисы | Текст статьи |
| Критик | Оценивает статью по баллу 1–10 | Тема + статья | Оценка + замечания |
| Оркестратор | Управляет порядком вызовов | — | Финальный результат |

## Стек

- **Python 3.9+**
- **OpenAI SDK** — для вызовов к OpenRouter (совместимый API)
- **OpenRouter** — доступ к бесплатным моделям (Llama 3, Mistral, Gemma)

## Быстрый старт

### 1. Зарегистрируйтесь на OpenRouter

Перейдите на [openrouter.ai](https://openrouter.ai), создайте аккаунт и получите API ключ в разделе **Keys**. Регистрация бесплатная, карта не нужна.

### 2. Установите зависимости

```bash
pip install -r requirements.txt
```

### 3. Задайте API ключ

**Windows (PowerShell):**
```powershell
$env:OPENROUTER_API_KEY = "sk-or-v1-..."
```

**Linux / macOS:**
```bash
export OPENROUTER_API_KEY="sk-or-v1-..."
```

Или создайте файл `.env` (он добавлен в `.gitignore`):
```
OPENROUTER_API_KEY=sk-or-v1-...
```

### 4. Запустите

```bash
# Интерактивный ввод темы
python main.py

# Тема из аргумента командной строки
python main.py "Квантовые вычисления"
python main.py "The impact of AI on the job market"
```

## Структура проекта

```
multi_agent_demo/
├── main.py            # Точка входа
├── orchestrator.py    # Оркестратор: управляет порядком агентов
├── claude_client.py   # Хелпер для вызова LLM через OpenRouter
├── agents/
│   ├── researcher.py  # Агент-Исследователь
│   ├── writer.py      # Агент-Писатель
│   └── critic.py      # Агент-Критик
└── requirements.txt
```

## Что демонстрирует проект

| Паттерн | Где смотреть |
|---------|-------------|
| Роль агента через `system` промпт | `agents/*.py` — у каждого свой `SYSTEM_PROMPT` |
| Передача контекста между агентами | `orchestrator.py` — output одного → input следующего |
| Оркестратор как Python-логика | `orchestrator.py` — обычный `if`, не LLM |
| Условная итерация | `if score < 7: writer.run(..., feedback=...)` |
| Смена модели для разных агентов | `call_agent(..., model="другая-модель")` |

## Бесплатные модели OpenRouter

Модели с суффиксом `:free` доступны без оплаты:

- `meta-llama/llama-3.1-8b-instruct:free`
- `mistralai/mistral-7b-instruct:free`
- `google/gemma-2-9b-it:free`

Изменить модель можно в `claude_client.py` (константа `FREE_MODEL`).
