# Идеи для развития проекта

## ✅ Реализовано: Вариант 1 — Параллельные агенты

Вместо одного Исследователя — три, работающих одновременно по разным аспектам темы.
Синтезатор объединяет результаты перед передачей Писателю.
Использует `concurrent.futures.ThreadPoolExecutor`.

```
           ┌─ Исследователь: Факты ────┐
Тема ──────┤─ Исследователь: Примеры ──┼──► Синтезатор ──► Писатель ──► Критик
           └─ Исследователь: Тренды ───┘
```

---

## Вариант 2 — Агенты с инструментами (Tool Use)

Агент может вызывать реальные функции: поиск в Wikipedia, запрос к API, калькулятор.
Оркестратор интерпретирует запросы агента и выполняет инструменты.
Это основа большинства реальных AI-систем.

```python
# Агент возвращает JSON с запросом инструмента:
{ "tool": "wikipedia_search", "query": "квантовые вычисления" }

# Оркестратор вызывает функцию и возвращает результат агенту
# Агент продолжает работу с реальными данными
```

**Что нужно добавить:**
- `tools/wikipedia.py` — поиск через Wikipedia API
- `tools/calculator.py` — вычисления
- В `orchestrator.py` — цикл: вызов агента → проверка tool_call → выполнение → возврат результата

## Вариант 3 — Мультиагентные дебаты

Два агента с противоположными позициями спорят о теме.
Судья слушает дебаты и выносит взвешенное решение.
Результат — сбалансированный текст с несколькими точками зрения.

```
Тема ──► Оппонент А (за) ──┐
                            ├──► 3 раунда дебатов ──► Судья ──► Итог
         Оппонент Б (против) ┘
```

**Что нужно добавить:**
- `agents/debater_pro.py` — аргументирует ЗА
- `agents/debater_con.py` — аргументирует ПРОТИВ
- `agents/judge.py` — оценивает дебаты и подводит итог
- `orchestrator_debate.py` — цикл раундов дебатов

---

## Вариант 4 — Иерархическая оркестрация

Главный оркестратор получает задачу и разбивает её на подзадачи для суб-оркестраторов.
Ближе всего к тому, как работает реальный Claude Code.

```
Главный оркестратор
├── Суб-оркестратор "Исследование" → [Факты, Источники, Статистика]
├── Суб-оркестратор "Создание"     → [Структура, Текст, Стиль]
└── Суб-оркестратор "Контроль"     → [Факты, Логика, Редактура]
```

**Что нужно добавить:**
- `orchestrators/research_team.py`
- `orchestrators/writing_team.py`
- `orchestrators/qa_team.py`
- `main_orchestrator.py` — координирует все три команды
